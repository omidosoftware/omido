## Critical Messaging Directives

### Technology Positioning
Do NOT center the homepage around technology names like:
- React
- Next.js
- C# / .NET
- Azure
- Stripe
- OpenAI

Technology may be referenced to support credibility, but it must not be the lead sales message.

Instead, the website should foreground themes such as:
- modern software
- secure implementation
- fast delivery
- maintainable systems
- scalable architecture
- AI integration where it adds value
- software built for growth

### Replace “Our Technology” Framing
If the current site has a section like “Onze technologie” with stack logos or tool labels, replace it with a stronger value-led framing such as:
- our priorities
- built for speed, security, and growth
- how we build
- what we optimize for
- modern software, properly built

The content should feel commercially relevant, not developer-internal.

### Pricing and Offer Framing
The site may communicate:
- trajectories starting from €995
- fast-scoped delivery options
- MVP or first SaaS versions often possible within 4 weeks for clearly scoped projects

But:
- do not make Omido feel cheap
- do not lead the homepage with bargain pricing
- do not make unrealistic blanket promises
- do not frame the company as a “cheap fast builder”

Position entry pricing as:
- a lower-friction way to start
- a validation step
- a scoped first project
- an accessible start to a more serious collaboration

## Constraints

Do not:
- produce generic section order without reasoning
- overuse gradients, glassmorphism, or decorative 3D
- add animation that harms performance or clarity
- create weak buzzword-heavy copy
- use fake proof
- make the homepage feel like a template
- sacrifice accessibility for style
- treat mobile as a scaled-down desktop
- use stack names as substitute for value proposition

## Motion and 3D Directive

Motion should be premium, restrained, and purposeful.

Use:
- Framer Motion for refined UI interaction
- GSAP for advanced sequencing and scroll choreography where justified
- Three.js / React Three Fiber / WebGL only when there is a clear strategic reason

Three.js should only be used if it adds one of these:
- a signature premium hero moment
- meaningful spatial storytelling
- distinctive brand depth
- high-end interactive communication of a system or product

Do not use advanced 3D:
- as generic decoration
- in multiple heavy sections by default
- in ways that weaken mobile performance
- in ways that distract from the core message

If using 3D, define:
- why it exists
- what it communicates
- what the fallback is
- how mobile performance is protected
