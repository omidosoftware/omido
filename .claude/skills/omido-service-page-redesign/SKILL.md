# Omido Service Page Redesign

## Purpose
Redesign service-page messaging and structure so services feel outcome-driven, premium, and specific.

## Use this skill when
- redesigning services pages
- improving offer clarity
- reducing generic service descriptions
- differentiating AI agents, web apps, SaaS, and integrations

## Focus
- what each service actually delivers
- who each service is for
- business outcomes
- trust building
- objection handling
- scope clarity
- CTA logic
- mobile readability
- proof opportunities per service

## Rules
- do not describe services in generic agency language
- do not rely on stack names to explain value
- do not keep vague labels without clear outcomes
- each service should feel buyable and understandable

## Required output
Provide:
1. current service-page weaknesses
2. recommended service-page structure
3. improved framing per service
4. better CTA strategy
5. proof recommendations
6. FAQ opportunities
7. final structure recommendation

## Standard
Each service page must help a serious client quickly understand value, fit, and next step.
