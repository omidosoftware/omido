# Omido Motion System

## Purpose
Define a motion system that increases perceived quality without harming clarity or performance.

## Use this skill when
- designing interaction behavior
- planning entrance animations
- planning hover states
- evaluating scroll effects
- refining premium polish

## Focus
- motion purpose
- timing discipline
- reveal choreography
- hover behavior
- click feedback
- section transitions
- page transitions
- reduced motion
- mobile-safe motion
- performance safety

## Required output
Provide:
1. motion principles
2. what should animate
3. what should not animate
4. timing/easing guidance
5. desktop vs mobile motion rules
6. reduced motion rules
7. implementation notes

## Standard
Motion must support UX, not compete with it.
