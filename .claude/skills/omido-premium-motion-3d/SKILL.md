# Omido Premium Motion and 3D

## Purpose
Decide how premium motion, GSAP, Framer Motion, and Three.js should be used strategically in the redesign.

## Use this skill when
- deciding motion stack
- planning premium interaction
- considering 3D or WebGL
- evaluating signature moments
- protecting mobile performance

## Focus
- when to use Framer Motion
- when to use GSAP
- when to use Three.js / React Three Fiber / WebGL
- hero signature moments
- scroll storytelling
- hover systems
- desktop vs mobile motion
- performance budgets
- accessibility and reduced motion
- fallback behavior

## Rules
- motion must have a job
- 3D must have a reason
- do not use 3D as random decoration
- do not weaken mobile performance for aesthetics
- prefer one strong premium moment over many average effects

## Required output
Provide:
1. recommended motion stack strategy
2. where Framer Motion should be used
3. where GSAP should be used
4. whether Three.js is justified
5. if Three.js is justified, where and why
6. mobile and fallback rules
7. implementation constraints
8. final recommendation

## Standard
Motion should increase perceived quality, not technical risk.
