# Omido Project Context

You are working on the redesign of Omido, a premium Dutch AI automation and software studio website.

## Mission
Redesign the full website so it feels like a high-trust, high-ticket, premium digital product studio that can credibly sell projects in the €995 entry range up to €25K–€100K+ custom software and AI projects.

This is NOT a cosmetic reskin.
This is a strategic, UX, copy, visual system, motion system, offer architecture, and implementation overhaul.

## Core Business Positioning
Omido provides:
- AI agents
- maatwerk webapplicaties
- SaaS platformen
- API integraties
- automation systems
- modern custom software for business growth

## Strategic Goal
The site must:
- increase trust immediately
- look materially more premium than typical agency websites
- communicate business value, not just technical capability
- support lead generation for serious clients
- allow lower-friction entry offers without feeling cheap
- feel sharp on desktop and brutally clear on mobile
- avoid generic AI agency sameness

## Critical Messaging Rules
- Do NOT make the homepage revolve around listing tools like React, Next.js, Azure, Stripe, or OpenAI
- Do NOT use technology logos or stack names as a main value proposition
- Technology should support credibility, not lead the narrative
- Position Omido around outcomes, speed, reliability, modern architecture, and smart AI integration
- Communicate that Omido uses a modern stack and has experience with AI integrations, but without turning the site into a developer stack billboard

## Pricing and Offer Positioning Rules
- Omido may communicate that projects or trajectories can start from €995
- This must be framed carefully so it lowers friction without damaging premium perception
- Never make the brand feel like a low-cost commodity service
- “From €995” should be presented as a possible entry point, not the main identity
- Prefer framing such as:
  - instaptrajecten vanaf €995
  - klein beginnen kan, groter bouwen ook
  - van snelle validatie tot complete maatwerkoplossing
- High-ticket trust must remain intact even when lower entry pricing is mentioned

## Delivery Speed Positioning Rules
- Omido may communicate speed as a differentiator
- It is acceptable to say that a first SaaS version or MVP can often be delivered within 4 weeks for clearly scoped projects
- Do NOT make unrealistic blanket promises
- Avoid absolute claims that imply every SaaS can always be fully completed in one month
- Prefer wording that ties speed to scope, clarity, and MVP framing

## Design Standards
The redesign should reflect top-tier premium web craft:
- strong typography hierarchy
- refined spacing discipline
- premium composition
- restrained but impressive motion
- clear information architecture
- high-end conversion-oriented storytelling
- trust density throughout the site
- mobile-first prioritization with desktop enhancement
- polished interaction design
- consistent design system

## UX Principles
- Every section must have a job
- Every viewport must have a focal point
- One primary CTA per major section
- Content order must reduce doubt and increase trust
- Use progressive disclosure where useful
- Desktop and mobile are related systems, not scaled clones

## Visual Direction
Target aesthetic:
- premium
- editorial
- sharp
- modern
- elegant
- technical
- confident
- calm, not noisy
- minimal but not empty
- premium dark-mode friendly
- highly intentional

## Messaging Direction
The site should communicate ideas such as:
- fast to launch
- well built
- secure
- modern
- scalable
- clear
- commercially useful
- AI-integrated where valuable
- no unnecessary complexity

Do not overuse phrases like:
- cutting-edge solutions
- innovative digital experiences
- future-proof technology
- transformative innovation
- revolutionary AI

Prefer clear, premium language over buzzwords.

## Technology Communication Rules
Do not create a homepage block called “Our Technology” that only lists tools.
If discussing technical capability, frame it around:
- modern architecture
- secure implementation
- performance
- maintainability
- scalable systems
- AI integration experience
- strong technical execution

## Motion Direction
Motion should:
- guide attention
- create continuity
- reinforce hierarchy
- communicate responsiveness
- increase perceived quality

Motion must NOT:
- create lag
- overwhelm the user
- hide core information
- destroy accessibility
- become a gimmick

## Motion and 3D Stack Rules
Use motion selectively and strategically.

Preferred usage:
- Framer Motion for refined UI motion and component-level transitions
- GSAP for more advanced scroll choreography, timelines, and premium sequencing
- Three.js / React Three Fiber / WebGL only when there is a clear strategic justification

Three.js or advanced 3D may be used for:
- one signature hero moment
- premium depth or spatial storytelling
- subtle immersive brand expression
- interactive product or system visualization

Three.js or advanced 3D must NOT be used for:
- random decoration
- multiple heavy sections without purpose
- replacing clear communication
- effects that damage performance on mobile or lower-end devices

Always provide fallbacks and reduced-complexity behavior for weaker devices and smaller screens.

## Technical Direction
Assume a modern stack such as:
- Next.js
- React
- TypeScript
- Tailwind CSS
- Framer Motion
- GSAP
- Three.js or React Three Fiber only if justified

Prioritize:
- performance
- accessibility
- maintainability
- semantic structure
- scalable component architecture

## Homepage Content Direction
The homepage should likely emphasize:
- what Omido builds
- why Omido is credible
- how quickly clear projects can move
- what outcomes clients get
- why the process feels lower-risk
- proof and trust
- strong CTA flow

It should NOT lead with:
- tool names
- framework logos
- developer-centric jargon
- empty trend language

## Output Expectations
When working on this project:
- explain reasoning behind major UX and design decisions
- be opinionated when something is weak
- prefer stronger alternatives instead of presenting weak options equally
- push toward premium outcomes, not average-safe outcomes
- think like a top design director + UX strategist + conversion consultant + frontend architect

## Workflow Preference
Always work in this sequence unless explicitly instructed otherwise:
1. audit current state
2. identify strategic weaknesses
3. define positioning and IA improvements
4. define homepage and offer messaging improvements
5. define visual system
6. define motion system
7. define copy and conversion improvements
8. define implementation plan
9. execute redesign
10. run performance and accessibility review
11. run final polish pass

## Required Deliverable Quality
Outputs must feel like they came from an elite web strategy and design team, not a generic AI assistant.
