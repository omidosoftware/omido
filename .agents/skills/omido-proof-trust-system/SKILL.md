# Omido Proof and Trust System

## Purpose
Increase the site’s credibility, proof density, and trust architecture so it can support both entry offers and premium projects.

## Use this skill when
- improving trust signals
- redesigning proof blocks
- planning testimonials and case studies
- improving conversion quality
- strengthening premium perception

## Focus
- proof density
- case study structure
- testimonials
- delivery credibility
- process clarity
- founder/studio trust
- guarantees or reassurance language
- FAQ trust support
- contact page trust
- why Omido feels lower-risk to buy from

## Rules
- no fake proof
- no shallow social proof
- no weak generic claims
- proof must support the offer and price positioning

## Required output
Provide:
1. current trust weaknesses
2. proof opportunities
3. recommended trust architecture
4. case study recommendations
5. testimonial guidance
6. process/reassurance recommendations
7. final trust system recommendation

## Standard
A premium website should make the buyer feel safer, clearer, and more confident.
